<br/>

A multi-agent AI assistant that manages full project-management workflows through natural conversation — task creation, updates, queries, and team coordination — backed by a two-stage router/responder architecture and a graph-native data layer.

| Category | Details |
|---|---|
| **Stack** | LangGraph, LangChain, FastAPI, Groq (llama-3.3-70b-versatile, llama-3.1-8b-instant), Neo4j, MongoDB, Node.js |
| **Scale** | Full-stack multi-agent system with cross-service API orchestration |
| **Performance** | Prompt footprint optimized from ~10,800 → ~7,500 tokens per call |
| **Security** | Confirmation-gate pattern for all mutating operations before execution |
| **Impact** | Final Year Project — production-style agentic PM assistant built with a 3-person team |
| **Repository** | *Private — available on request* |

**Engineering Highlights**
- Designed a **two-stage router/responder architecture** with a semantic router layered on deterministic Python pre-checks to reduce unnecessary LLM invocations.
- Implemented a **preview-then-commit pattern** to prevent double LLM calls on confirmable actions.
- Built a custom `SanitizedChatGroq` subclass to resolve LangChain/Groq tool-call parsing failures.
- Owned the full stack — `agent.py`, `tools.py`, `router_prompt.py`, `responder_prompt.py`, `multi_step_nodes.py` — alongside Node.js route/controller integration.
- Resolved persistent Cypher query mismatches and cross-stack API misalignment between FastAPI and Node.js services.
